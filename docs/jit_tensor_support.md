# JIT Tensor Support Documentation

## Overview
The HVM JIT now provides first‑class support for **tensor metadata** and **floating‑point vector arithmetic**. This enables generated code to efficiently operate on tensors of arbitrary rank and element type without relying on fragile raw memory offsets.

## Runtime Helpers
Three new JIT‑exposed helper functions are registered in the JIT symbol table:

- `jit_tensor_rank(HooTensor* t) -> int64_t`
- `jit_tensor_element_type(HooTensor* t) -> int64_t`
- (internal) `_F_hoo_Tensor_*` wrappers used by the code generator.

These helpers query the underlying `hoo_tensor` C++ API (`hoo_TensorRank`, `hoo_TensorElementType`) and return the values as 64‑bit integers that the JIT can consume directly.

## Symbol Table Registration
In `src/hvm/HVMJIT.cpp` the helpers are added:
```cpp
jit->addSymbol("jit_tensor_rank", (void*)jit_tensor_rank);
jit->addSymbol("jit_tensor_element_type", (void*)jit_tensor_element_type);
```
The JIT code generator now emits calls to these helpers when it needs the tensor's rank or element type, replacing the previous hard‑coded `LD_D` loads.

## Memory Addressing
The `memAddr` lambda was enhanced to differentiate between:
- **VM‑relative offsets** (default) – addressed via `state->memory`.
- **Raw host pointers** – any address ≥ 1 000 000 000 is treated as a direct host address and passed unchanged to the generated code.
This allows JIT code to work with tensors that are allocated in host memory (e.g., via `malloc`).

## Vector Arithmetic Support
The JIT now supports element‑wise vector arithmetic for floating‑point types (`float` and `double`).

- The `vtype` register is set based on the tensor's element type (`2` for `float`, `9` for `double`).
- `VSETVL` is emitted with the correct `vtype`.
- The runtime helper `hooc_hvm_vector_arith` performs the arithmetic using the element type from `vtype`.

Supported operations include addition, subtraction, multiplication, and division on vectors of equal length.

## Interpreter Changes
The interpreter cases `VECTOR_MEM` and `VECTOR_ARITH` now correctly handle host‑mapped memory addresses and dispatch to the updated `hooc_hvm_vector_arith` implementation, which selects the appropriate arithmetic path based on the `vtype` register.

## Usage Example
```cpp
// Assume tensorPtr points to a HooTensor allocated in host memory.
int64_t rank = jit_tensor_rank(tensorPtr);
int64_t elemType = jit_tensor_element_type(tensorPtr);
// JIT code can now safely use `rank` and `elemType`.
```

All unit tests, including `HooTensorJitTest.TensorF64ElementwiseArithmetic`, now pass, confirming correct behavior for double‑precision element‑wise arithmetic.
